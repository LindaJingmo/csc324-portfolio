Project Journal Log

April 27, 2026 · 3 min · Linda Jing

February 3, 2026#

Our group agreed on the basic technology direction for the project. We planned to use React/Vite for the frontend, Node or Python for backend exploration, SQL for the database, and GitHub for code sharing.

February 4, 2026#

I conducted an observation and interview for needfinding. I organized the notes into a file so the group could discuss user needs and possible planner features in our next meeting.

February 5, 2026#

We discussed specialization and divided responsibilities. The frontend would focus on React UI and planner interaction, while I began thinking about backend/database structure and how course data could support the planner.

February 12, 2026#

We refined our project direction around a four-year course planner. I focused on understanding what kinds of information the database would need, such as majors, courses, requirements, and possible prerequisite relationships.

February 20, 2026#

I started researching how to represent academic requirements in SQL. I realized that requirements were not all the same type, so the database would need a flexible structure instead of one table per major.

February 28, 2026#

The group continued frontend planning while I worked on early database design ideas. We used simpler data formats first so the frontend could keep moving while the SQL structure was still being developed.

March 7–23, 2026#

Spring break. I did not make major project changes during this period, but I continued thinking about how SQL requirement data would eventually connect to the frontend planner.

March 25, 2026#

After spring break, I returned to the database design and started turning the requirement model into SQL tables. I focused on courses, majors, requirement blocks, and course options.

April 2, 2026#

I worked on the core SQL schema and began writing major seed files. This helped me practice medium-scale abstraction because different majors could use the same requirement table structure.

April 10, 2026#

I continued adding and correcting major requirement data. I focused on making the SQL files repeatable so the database could be rebuilt without manually inserting every table again.

April 17, 2026#

I worked on linking the SQL data to the React planner through the backend/API layer. This step was difficult because the frontend, backend, and PostgreSQL database all had to run correctly at the same time.

April 22, 2026#

I debugged requirement-checking problems where some courses appeared in the planner but did not correctly satisfy requirements. This helped me separate frontend display issues from SQL data-modeling issues.

April 26, 2026#

I tested the SQL-backed planner by rebuilding the database, running the backend, and checking the frontend in the browser. I also documented how my SQL schema, architecture, and debugging process connect to the course competencies.

April 28, 2026#

I participated in code review and discussed requirement-checking logic with my partners. We went back and forth about how the planner should interpret major requirements, prerequisite rules, and user-selected courses.

April 30, 2026#

During this week, our group prepared and practiced the internal presentation. I reviewed the parts of the project I was responsible for, especially the SQL-backed requirement structure and how it connected to the planner interface.

May 7, 2026#

I changed and polished code for the external presentation. I focused on making the planner easier to demonstrate, improving the visible flow of the interface, and making sure the major requirement logic could be explained clearly.
