Self-Learning: SQL to React Tutorial

April 26, 2026 · 2 min · Linda Jing

I wrote a tutorial connecting SQL to React: Tutorial: Linking SQL to Reacts

Describe the technology and how it fits into your project#

My tutorial covers linking SQL to React through a backend API using Express.js and PostgreSQL. As I wrote in the tutorial, “React does not connect directly to SQL — instead, React communicates with a backend server, and the backend server communicates with the SQL database.” In our project, this pattern was how the requirement_blocks schema I designed actually reached the React frontend so users could see their degree progress.

Evaluate the tool — strengths and weaknesses#

The biggest strength is clear separation of concerns: SQL stores data, Express controls access, React displays it. This made debugging much easier. The weakness is setup overhead — maintaining three layers is costly for a small team. Tools like Supabase offer a more direct path, but they can’t handle the complex SQL queries our requirement audit logic needed. It is our logic that required using this tool.

One sticking point your team should know#

Environment variables. As I noted in the tutorial, the database URL “should be stored in an environment variable, not directly written in the code.” Early on, team members ran the backend with the wrong or missing DATABASE_URL, causing silent connection failures. The fix: create a .env file, add it to .gitignore immediately, and document all required variables in the README before anyone else clones the repo.
