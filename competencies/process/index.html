Process

April 26, 2026 · 3 min · Linda Jing

1. What software engineering practices did your group employ to manage, distribute, and execute on their work?#

Our group divided the work by major parts of the system. I was responsible for the SQL side, including the database structure, major seed files, and linking SQL data to the planner. Drilon and Nahom were mainly responsible for the frontend and UI design, including the React planner interface and how users interact with courses, semesters, and requirements.

We also used an incremental development process. Before we fully figured out how to connect React to SQL, we used JSON as an intermediate data source. This allowed the frontend to keep developing and testing while I worked on the SQL structure and database-linking problem. Once the SQL connection became more complete, the SQL data could start replacing the temporary JSON data.

Even though responsibilities were divided, we still debugged across roles. Drilon and Nahom understood that the backend and database connection was a big part of making the frontend work, so they helped me debug when problems involved multiple layers.

2. In your estimation, how well did your group follow through on using these processes throughout the semester?#

I think our group followed this process fairly well. The division of responsibilities helped us make progress because each person had a clear area to focus on. The JSON intermediate step was especially helpful because frontend work did not have to stop while the SQL connection was still unfinished.

At the same time, our process was not perfect. The SQL, backend connection, and frontend UI were closely connected, so some tasks could not stay completely separate. For example, if a requirement did not update correctly, the issue could have been in the SQL data, the connection layer, or the React frontend. Because of that, we often had to debug together instead of only working inside our own assigned areas.

3. What, if anything, got in the way of integrating these processes into your workflow?#

The biggest challenge was the technical complexity of connecting multiple layers of the project. The frontend, JSON data, SQL database, backend API connection, and local database setup all had to work together. When something broke, it was not always obvious which layer caused the problem.

Another challenge was that we did not fully figure out the SQL-to-React connection at the beginning. This could have blocked frontend development, but using JSON as an intermediate step helped reduce that problem. It gave Drilon and Nahom a stable data source for frontend testing while I continued working on the SQL design and linking process.
