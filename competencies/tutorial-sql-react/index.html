Tutorial: Linking SQL to React

April 26, 2026 · 3 min · Linda Jing

React applications often need data that can be saved, updated, and reused. A SQL database is useful for this because it stores structured data in tables. However, React does not connect directly to SQL. Instead, React communicates with a backend server, and the backend server communicates with the SQL database.

The general structure looks like this:

1. Create a SQL Table#

First, create a table in your SQL database. For example, this table stores a list of items:

CREATE TABLE items (
  id SERIAL PRIMARY KEY,
  name TEXT NOT NULL,
  description TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Then insert some sample data:

INSERT INTO items (name, description) VALUES
  ('Notebook', 'A paper notebook'),
  ('Pen', 'A blue ink pen');

2. Create a Backend Server#

React should not access the database directly, so a backend server is needed. One common option is Express.js.

import express from "express";

const app = express();
const PORT = 3001;

app.use(express.json());

app.listen(PORT, () => {
  console.log(`Server running on port ${PORT}`);
});

This creates a server that can receive requests from React.

3. Connect the Backend to SQL#

For PostgreSQL, the pg library can connect the backend server to the database.

import pg from "pg";

const { Pool } = pg;

const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
});

The database URL should be stored in an environment variable, not directly written in the code.

4. Create an API Route#

Next, create an API route that reads data from SQL and returns it as JSON.

app.get("/api/items", async (req, res) => {
  try {
    const result = await pool.query("SELECT * FROM items ORDER BY id");
    res.json({ items: result.rows });
  } catch (error) {
    res.status(500).json({ error: "Database query failed" });
  }
});

When React requests /api/items, the backend runs the SQL query and sends back the result.

5. Fetch the API Data in React#

In React, use fetch() inside useEffect() to load the data when the page opens.

import { useEffect, useState } from "react";

function App() {
  const [items, setItems] = useState([]);

  useEffect(() => {
    async function loadItems() {
      const response = await fetch("/api/items");
      const data = await response.json();
      setItems(data.items);
    }

    loadItems();
  }, []);

  return (
    <div>
      <h1>Items</h1>
      {items.map((item) => (
        <div key={item.id}>
          <h2>{item.name}</h2>
          <p>{item.description}</p>
        </div>
      ))}
    </div>
  );
}

export default App;

React does not know the SQL query. It only receives JSON and displays it.

6. Use a Development Proxy#

If the frontend and backend run on different ports, a proxy can make requests easier. For example, in Vite:

export default {
  server: {
    proxy: {
      "/api": "http://localhost:3001",
    },
  },
};

Now React can call fetch("/api/items") instead of writing the full backend address.
